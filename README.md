# Next.js 게시판 프로젝트 진행 가이드

## 1. 프로젝트 목표

이번 과제의 목표는 Next.js를 이용하여 간단한 게시판 서비스를 구현하고, 이를 관리할 수 있는 관리자 기능까지 포함한 웹 애플리케이션을 로컬 환경에서 실행하는 것이다.

단순히 화면을 만드는 것에 그치지 않고, Next.js가 제공하는 라우팅, 상태 관리, 인증, API 기능 등을 단계적으로 경험하면서 프론트엔드 개발에 필요한 기본 역량을 익히는 데 목적이 있다.

최종적으로 구현해야 하는 기능은 다음과 같다.

* 게시글 목록 조회
* 게시글 상세 조회
* 게시글 작성
* 게시글 수정
* 게시글 삭제
* 관리자 로그인
* 관리자 게시글 관리
* 관리자 페이지 접근 제어

---

## 2. 1단계 : 프로젝트 생성 및 기본 라우팅

먼저 Next.js 프로젝트를 생성한다.

TypeScript와 App Router를 사용하여 프로젝트를 구성하고, 기본적인 페이지 이동 구조를 만든다.

구현 대상 페이지는 다음과 같다.

* 홈 화면 (/)
* 게시판 목록 (/board)
* 관리자 페이지 (/admin)

이 단계에서는 Next.js의 폴더 기반 라우팅 구조를 이해하는 것이 중요하다.

완료 기준은 다음과 같다.

* 프로젝트가 정상적으로 실행된다.
* 메뉴를 통해 각 페이지로 이동할 수 있다.
* App Router 구조를 이해한다.

---

## 3. 2단계 : 게시글 목록 구현

Mock 데이터를 이용하여 게시판 목록 화면을 구현한다.

게시글 데이터는 배열 형태로 직접 작성하여 사용한다.

목록 화면에는 다음 정보가 표시되어야 한다.

* 제목
* 작성자
* 작성일

이 과정에서 다음 내용을 학습한다.

* 배열 렌더링
* map 함수 활용
* key 속성 사용
* 컴포넌트 분리
* Props 전달

완료 기준은 다음과 같다.

* 게시글 목록이 정상적으로 출력된다.
* 각 게시글을 클릭하면 상세 페이지로 이동할 수 있다.

---

## 4. 3단계 : 게시글 상세 페이지 구현

동적 라우팅을 이용하여 게시글 상세 페이지를 구현한다.

예를 들어 다음과 같은 URL 구조를 사용한다.

* /board/1
* /board/2

URL에 포함된 게시글 ID를 이용하여 해당 게시글 정보를 화면에 표시한다.

이 단계에서는 다음 내용을 학습한다.

* Dynamic Route
* params 사용법
* 데이터 조회 방식

완료 기준은 다음과 같다.

* 게시글 ID에 따라 서로 다른 상세 화면이 출력된다.

---

## 5. 4단계 : 게시글 작성 기능 구현

게시글 작성 화면을 구현한다.

입력 항목은 다음과 같다.

* 제목
* 내용
* 작성자

작성 버튼을 누르면 게시글이 생성되도록 구현한다.

우선 React의 기본 상태 관리(useState)를 이용하여 구현한 뒤, 가능하다면 React Hook Form을 활용하여 리팩토링한다.

이 단계에서 학습하는 내용은 다음과 같다.

* Controlled Component
* Form 처리
* Submit 이벤트 처리
* React Hook Form 사용법

완료 기준은 다음과 같다.

* 게시글을 작성할 수 있다.
* 작성 후 목록 화면으로 이동한다.

---

## 6. 5단계 : LocalStorage 연동

현재 메모리에서만 관리되는 게시글 데이터를 LocalStorage에 저장하도록 변경한다.

이를 통해 새로고침 이후에도 게시글이 유지되도록 구현한다.

학습 내용은 다음과 같다.

* localStorage.getItem
* localStorage.setItem
* JSON.parse
* JSON.stringify

완료 기준은 다음과 같다.

* 작성한 게시글이 브라우저를 새로고침해도 유지된다.

---

## 7. 6단계 : 수정 및 삭제 기능 구현

게시판의 CRUD 기능을 완성한다.

추가해야 하는 기능은 다음과 같다.

* 게시글 수정
* 게시글 삭제

수정 화면에서는 기존 데이터를 불러와 수정할 수 있어야 하며, 삭제 후에는 목록 화면으로 이동하도록 구현한다.

이 단계에서 학습하는 내용은 다음과 같다.

* 배열 데이터 업데이트
* 기존 데이터 활용
* 화면 이동 처리

완료 기준은 다음과 같다.

* 게시글 생성, 조회, 수정, 삭제가 모두 가능하다.

---

## 8. 7단계 : 관리자 로그인 구현

간단한 관리자 인증 기능을 구현한다.

별도의 서버 인증 없이 고정된 계정을 사용한다.

예시 계정은 다음과 같다.

아이디 : admin
비밀번호 : 1234

로그인 성공 시 브라우저 저장소에 로그인 상태를 저장한다.

이 단계에서 학습하는 내용은 다음과 같다.

* 인증 흐름 이해
* 로그인 상태 관리
* 로그아웃 처리

완료 기준은 다음과 같다.

* 로그인 기능이 정상 동작한다.
* 로그인 여부를 화면에서 확인할 수 있다.

---

## 9. 8단계 : 관리자 게시글 관리 기능 구현

관리자 전용 게시글 관리 화면을 구현한다.

관리자는 일반 사용자보다 더 많은 권한을 가진다.

구현 기능은 다음과 같다.

* 게시글 목록 조회
* 게시글 수정
* 게시글 삭제
* 게시글 검색

또한 사용자 화면과 관리자 화면에서 공통으로 사용하는 컴포넌트를 분리하여 재사용하도록 한다.

완료 기준은 다음과 같다.

* 관리자 로그인 후 게시글을 관리할 수 있다.
* 일반 사용자는 해당 기능에 접근할 수 없다.

---

## 10. 9단계 : 관리자 접근 제어 적용

Next.js Middleware를 이용하여 관리자 페이지 접근을 보호한다.

관리자 인증이 되지 않은 상태에서 관리자 URL에 직접 접근할 경우 로그인 페이지로 이동하도록 구현한다.

이 단계에서 학습하는 내용은 다음과 같다.

* Middleware 동작 방식
* 접근 제어
* 리다이렉트 처리

참고로 실제 Middleware에서는 LocalStorage에 접근할 수 없으므로, 가능하다면 Cookie 기반으로 인증 방식을 변경하여 구현해보는 것을 권장한다.

완료 기준은 다음과 같다.

* 비로그인 상태에서 관리자 페이지 접근이 차단된다.

---

## 11. 10단계 : Next.js API Route 적용

기존의 LocalStorage 기반 구조를 Next.js API Route 기반으로 변경한다.

게시글 데이터를 API를 통해 조회하고 수정하도록 구현한다.

구현 대상은 다음과 같다.

* 게시글 목록 조회 API
* 게시글 상세 조회 API
* 게시글 생성 API
* 게시글 수정 API
* 게시글 삭제 API

이 단계에서 학습하는 내용은 다음과 같다.

* GET
* POST
* PUT
* DELETE
* fetch를 이용한 API 호출

완료 기준은 다음과 같다.

* 프론트엔드와 API 간 통신이 정상적으로 이루어진다.

---

## 12. 11단계 : 구조 개선 및 리팩토링

마지막으로 프로젝트 구조를 정리한다.

기능 단위로 파일을 분리하고 공통 요소를 추출하여 유지보수가 쉬운 구조로 개선한다.

다음 내용을 고려한다.

* 타입 분리
* 공통 컴포넌트 구성
* 기능 단위 폴더 구성
* 유효성 검사 적용
* 에러 처리
* 로딩 처리

가능하다면 Zod를 이용한 입력값 검증도 적용해본다.

완료 기준은 다음과 같다.

* 프로젝트 구조가 명확하게 정리되어 있다.
* 기능별 책임이 분리되어 있다.
* 유지보수가 가능한 형태로 개선되어 있다.

---

## 과제 진행 권장 일정

Mission 1

* 프로젝트 생성
* 기본 라우팅
* 게시글 목록 및 상세 구현

Mission 2

* 게시글 작성
* 수정 및 삭제
* LocalStorage 저장

Mission 3

* 관리자 로그인
* 관리자 게시글 관리

Mission 4

* Middleware 적용
* API Route 적용
* 구조 리팩토링

위 과정을 모두 완료하면 단순한 React 화면 구현을 넘어, Next.js를 활용한 실제 서비스 구조를 경험할 수 있으며, 프론트엔드 개발자가 알아야 할 기본적인 풀스택 흐름까지 이해할 수 있게 된다.
